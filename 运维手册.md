# 舟山锚地供油指数看板 — 运维手册

## 项目概览

一个自动更新的锚地天气看板，展示舟山四个锚地（条帚门、虾峙门外、马峙、秀山东）的 MIO 供油指数精细化预报。

```
│  舟山海洋气象台  │────▶│  你的 Mac 电脑   │────▶│  GitHub 仓库       │────▶│  GitHub Pages    │
│  zs121.com.cn  │     │  定时抓数据      │     │  存代码+数据       │     │  托管网页         │
│  (API数据源)    │     │  crontab 自动跑  │     │  自动触发部署      │     │  用户访问看板     │
└──────────────┘     └───────────────┘     └──────────────────┘     └────────────────┘
```

## 运行逻辑

### 整体流程

1. **Mac crontab** 每天 10:00 和 16:00 自动触发
2. **Python 脚本**（`scripts/update_data.py`）请求舟山海洋气象台 API，抓取四个锚地的预报数据
3. 脚本将数据写入 `data/latest.json` 和 `data/data.js`
4. **git commit + push** 将新数据推送到 GitHub
5. **GitHub Pages** 检测到 main 分支更新，自动重新部署网页
6. 用户访问 `mio.zsagent01.com` 看到最新数据

### 自动更新不需要你做任何操作

电脑开机、连着网，就会自动跑。你什么都不用管。

### 什么情况下不会更新

- 电脑关机或合盖休眠（下次开机后到时间点会自动恢复）
- 断网（脚本会重试3次，全失败则跳过本次）
- zs121.com.cn 服务器宕机（少见，等下次自动恢复）

## 日常操作

### 查看是否正常运行

```bash
cat /tmp/mio-update.log
```

正常输出长这样：
```

时间: 2026-04-25 10:00:01
抓取 条帚门锚地 ...
  ✓ 成功，24 条时段数据
抓取 虾峙门外锚地 ...
  ✓ 成功，24 条时段数据
...
全部完成！
```

### 手动触发一次更新

```bash
cd ~/Documents/网站制作 && python3 scripts/update_data.py && git add data/ && git commit -m "🔄 手动更新" && git push
```

### 查看定时任务配置

```bash
crontab -l
```

### 修改更新时间

```bash
crontab -e
```

用 vi 编辑器打开（按 `i` 进入编辑模式，改完按 `Esc`，输入 `:wq` 保存退出）。

时间格式：`分 时 * * *`
- `0 10` = 每天 10:00
- `0 16` = 每天 16:00
- `30 8` = 每天 08:30
- `0 8,12,17` = 每天 8:00、12:00、17:00（三次）

### 临时停止自动更新

```bash
crontab -r
```

### 恢复自动更新

找 Cola 帮你重新配，或者手动：
```bash
crontab -e
```
粘贴以下内容保存：
```
# 舟山锚地MIO数据更新 — 每天 10:00 和 16:00
PATH=/usr/local/bin:/usr/bin:/bin
0 10 * * * cd /Users/qianlingbo/Documents/网站制作 && /usr/bin/python3 scripts/update_data.py && /usr/bin/git add data/ && /usr/bin/git diff --cached --quiet || (/usr/bin/git commit -m "🔄 更新MIO数据 $(date '+\%Y-\%m-\%d \%H:\%M')" && /usr/bin/git push) >> /tmp/mio-update.log 2>&1
0 16 * * * cd /Users/qianlingbo/Documents/网站制作 && /usr/bin/python3 scripts/update_data.py && /usr/bin/git add data/ && /usr/bin/git diff --cached --quiet || (/usr/bin/git commit -m "🔄 更新MIO数据 $(date '+\%Y-\%m-\%d \%H:\%M')" && /usr/bin/git push) >> /tmp/mio-update.log 2>&1
```

## 故障排查

### 网页显示旧数据

1. 浏览器强制刷新：`Cmd + Shift + R`
2. 检查日志：`cat /tmp/mio-update.log`，看最后一次是否成功
3. 检查 GitHub 是否收到推送：https://github.com/qianlingbo/zhoushan-anchorage-mio/commits/main

### 日志显示抓取失败

```
⚠ 失败: HTTPSConnectionPool ... timed out
```
→ zs121.com.cn 暂时不可达，等下次自动重试即可。如果连续多天失败，可能是对方改了接口。

### git push 失败

```
error: failed to push some refs
```
→ 远程有更新，执行：
```bash
cd ~/Documents/网站制作 && git pull --rebase && git push
```

如果有冲突：
```bash
cd ~/Documents/网站制作 && git stash && git pull && git stash drop && python3 scripts/update_data.py && git add data/ && git commit -m "🔄 手动更新" && git push
```

### 网页完全打不开（404）

1. 检查 GitHub Pages 是否开启：仓库 Settings → Pages → Source 应为 `main` 分支
2. 检查 CNAME 文件是否存在：`cat ~/Documents/网站制作/CNAME`，应该输出 `mio.zsagent01.com`
3. 检查 Cloudflare DNS 的 `mio` CNAME 记录是否指向 `qianlingbo.github.io`

### 定时任务没执行

1. 确认电脑没有在休眠：合盖期间 crontab 不运行
2. 确认 crontab 还在：`crontab -l`
3. macOS 可能会弹窗要求授权 cron 的磁盘访问权限 → 去 系统设置 → 隐私与安全性 → 完全磁盘访问权限，检查 `/usr/sbin/cron` 是否在列表中
